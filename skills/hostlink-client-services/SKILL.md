# client-services

> **PREREQUISITE:** Run `hostlink set-token <token>` to authenticate before using any command.

Manage services subscribed by a client (`ClientService`). A client can have multiple client services. Client services are the source records used when creating invoices — each `invoice-item` (PaymentPeriod) links back to a `clientservice_id`.

```bash
hostlink client-services <command> [flags]
```

## Commands

| Command | Description |
|---------|-------------|
| `hostlink client-services list` | List client services |
| `hostlink client-services get <id>` | Get a client service by ID |
| `hostlink client-services add` | Add a new client service |
| `hostlink client-services update <id>` | Update a client service |
| `hostlink client-services delete <id>` | Delete a client service |

---

## list

```bash
hostlink client-services list [flags]
```

| Flag | Description | Default |
|------|-------------|---------|
| `-c, --client <id>` | Filter by client ID | — |
| `--service <id>` | Filter by service ID | — |
| `-s, --search <text>` | Filter by name (contains) | — |
| `-l, --limit <n>` | Max records | `50` |
| `-o, --offset <n>` | Skip records | `0` |
| `--json` | Output as JSON | — |

```bash
hostlink client-services list --client 42
hostlink client-services list --client 42 --json
hostlink client-services list --search "hosting"
```

### Sample output

```
[10] Web Hosting | client:42 | $500/1mo | joined:2023-01-01 | paid to:2026-03-31
[11] Domain Renewal | client:42 | $200/12mo | joined:2023-01-01 | paid to:2026-01-31 [no-invoice]
```

## get

```bash
hostlink client-services get <id> [--json]
```

Returns full details including last invoice and last payment period.

```bash
hostlink client-services get 10
hostlink client-services get 10 --json
```

### Key computed fields (read-only)

| Field | Description |
|-------|-------------|
| `paidPeriodTo` | Date up to which payment has been received |
| `paymentPeriodTo` | Next payment period end date |
| `revenue` | Total revenue generated by this service |
| `lastInvoice` | Most recent invoice for this service |
| `lastPaymentPeriod` | Most recent PaymentPeriod record |

## add

```bash
hostlink client-services add --client-id <id> [flags]
```

| Flag | Description |
|------|-------------|
| `-c, --client-id <id>` | Client ID *(required)* |
| `--service-id <n>` | Service ID |
| `--domain-id <n>` | Linked domain ID |
| `--quotation-id <n>` | Linked quotation ID |
| `--quotationitem-id <n>` | Linked quotation item ID |
| `--name <value>` | Service name |
| `--title <value>` | Display title |
| `--join-date <date>` | Start date (YYYY-MM-DD) |
| `--end-date <date>` | End date (YYYY-MM-DD) |
| `--unit-price <n>` | Unit price |
| `--unit-month <n>` | Billing cycle in months |
| `--unit-quantity <n>` | Quantity |
| `--discount <n>` | Discount percentage |
| `--remark <value>` | Remark |
| `--clientservice-type <n>` | Service type (int) |
| `--renew-day <n>` | Renewal day |
| `--server-id <n>` | Server ID |
| `--no-invoice` | Exclude from invoicing |

```bash
hostlink client-services add --client-id 42 --name "Web Hosting" --unit-price 500 --unit-month 1 --join-date "2026-04-01"
```

Returns the new `clientservice_id` on success.

## update

```bash
hostlink client-services update <id> [flags]
```

Supports all the same flags as `add`, plus termination fields:

| Flag | Description |
|------|-------------|
| `--termination-date <date>` | Termination date (YYYY-MM-DD) |
| `--termination-no <n>` | Termination notice number |
| `--termination-reason <value>` | Termination reason |
| `--expect-termination-date <date>` | Expected termination date (YYYY-MM-DD) |

```bash
hostlink client-services update 10 --unit-price 600 --remark "price updated"
hostlink client-services update 10 --termination-date "2026-06-30" --termination-reason "client closed"
```

## Relationship to Invoices

```
Client
  └── ClientService (hostlink client-services)
        └── PaymentPeriod / invoice-item (hostlink invoice-items)
              └── Invoice (hostlink invoices)
```

When creating an invoice item with `hostlink invoice-items add`, use `--clientservice-id` to link the item to the correct `ClientService`. The `description` field on the invoice item is automatically derived from the linked `ClientService`.
